## ⚙️ Key Principles of GitHub Actions

To give context for where things might be going wrong:

- Workflow files must be in the `.github/workflows/` directory and have `.yml` (or `.yaml`) extension to be recognized. ([labex.io](https://labex.io/tutorials/git-github-actions-introduction-and-setup-633882?utm_source=openai))  
- Any processing of `.md` files (rendering, deploying, parsing, linking) must be explicitly called in the workflow file — e.g. steps that copy, build, generate pages, etc. Just having `.md` files in the repo does nothing by itself for Actions.  
- If using GitHub Pages or a static site generator, there might be additional constraints: placement (must be in `docs/` or root depending on settings), whether front-matter is present, layout/theme compatibility, etc.

## Common reasons `.md` files don‚Äôt show up/usefully

Here are frequent causes encountered when `.md` files exist but don‚Äôt actively show up where you expect them:

- **No link/reference**: If nothing in your UI (site, README, Docs) includes the links, they won‚Äôt appear anywhere visible beyond the file explorer.
- **Wrong folder for GitHub Pages**: Pages often serves content only from a folder like `docs/` or the root, *depending on settings*. If your `.md` files are outside that folder or not part of the build source, they won‚Äôt be published.  
- **No front-matter / layout**: When using static site generators (Jekyll, Hugo, etc.), Markdown files often need front-matter (YAML header) to get processed properly. Without it, they might be ignored or shown with no layout.  
- **Wrong triggering workflows**: If you rely on CI, GitHub Actions, etc. to process the `.md` files, there must be workflow `.yml` definitions that explicitly reference them. `.sh` files alone won‚Äôt trigger automatically unless called inside `.yml` workflows.  
- **File naming or casing issues**: Case-sensitive paths, special characters, spaces, etc. can cause issues where files don‚Äôt resolve or are overlooked in site generation.  
- **Permissions or `.gitignore` rules**: Though less likely, sometimes `.md` files are accidentally ignored or not committed; check `.gitignore` or if the files were staged/committed properly.
